# 项目复盘模板：从功能到面试表达

项目复盘不是简单列功能，而是把“我做了什么”变成“我为什么这样做、我解决了什么问题”。

## 背景

先用一两段说明项目要解决的问题。不要一上来堆技术栈，读者需要先知道这个项目为什么存在。

## 技术栈

可以按层次写：

- 前端：Vue、React 或原生页面
- 后端：Spring Boot、MyBatis、JWT
- 数据库：MySQL、Redis
- 部署：Docker、Nginx、云服务器

## 核心功能

选择 3 到 5 个真正能体现复杂度的功能，不要把所有 CRUD 都写一遍。

```java
@Transactional
public void createOrder(CreateOrderCommand command) {
    // 校验库存、创建订单、扣减库存需要处在一个一致性边界里。
}
```

## 难点和解决方案

一个好的难点通常长这样：

- 有明确业务后果
- 有多个可选方案
- 你能解释为什么最终选择这个方案

## 面试表达

最后把项目压缩成 60 秒版本：

> 我做的是一个面向某类用户的系统，核心问题是某某。我主要负责某几个模块，其中最有挑战的是某某，因为它涉及一致性、性能或权限边界。我最后通过某某方案解决，并用某某方式验证。
