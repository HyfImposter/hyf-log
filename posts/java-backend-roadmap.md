# Java 后端学习路线第一版

这篇文章先把学习路线写成一个可以执行的版本。重点不是“看过很多东西”，而是每一段学习都能产出代码、笔记和项目证据。

## 阶段一：Java 基础

- 面向对象、集合、泛型、异常
- I/O、反射、注解
- 多线程基础和线程池

这一阶段的输出可以是一个小型命令行项目，例如待办事项、文件索引器或简单记账工具。

## 阶段二：数据库和缓存

- MySQL 表设计、索引和事务
- Redis 常用数据结构
- 缓存穿透、击穿、雪崩

学习数据库时，不要只背概念。每个概念最好配一段 SQL 和一个实际业务场景。

## 阶段三：Spring Boot 项目

项目文章可以重点写这几个问题：

1. 接口如何设计？
2. 数据表为什么这样拆？
3. 哪些地方用了事务？
4. 遇到过什么性能或一致性问题？

## 自测问题

- 线程池为什么不建议直接用 `Executors` 创建？
- B+ 树索引适合什么查询？
- Redis 为什么常用作缓存？
